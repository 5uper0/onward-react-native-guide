useTeams: Загрузка + Фильтрация

Один хук объединяет fetch, filters и computed filteredTeams. Аналог ViewModel в iOS с @Published + Combine.

Hooks
// src/hooks/useTeams.ts export function useTeams(): UseTeamsReturn { const [teams, setTeams] = useState<FootballTeam[]>([]); const [isLoading, setIsLoading] = useState(true); const [error, setError] = useState<string | null>(null); const [filters, setFilters] = useState<Filters>(INITIAL_FILTERS); const loadTeams = useCallback(async () => { setIsLoading(true); setError(null); try { const allTeams = await fetchAllTeams(); setTeams(allTeams); } catch (err) { const message = err instanceof Error ? err.message : 'Failed to load teams'; setError(message); } finally { setIsLoading(false); } }, []); useEffect(() => { loadTeams(); }, [loadTeams]); const filteredTeams = useMemo(() => { let result = [...teams]; if (filters.search.trim()) { const q = filters.search.toLowerCase().trim(); result = result.filter(t => t.name.toLowerCase().includes(q)); } if (filters.minValue !== null) result = result.filter(t => t.estimated_value_numeric >= filters.minValue!); if (filters.minTitles !== null) result = result.filter(t => t.number_of_league_titles_won >= filters.minTitles!); result.sort((a, b) => b.estimated_value_numeric - a.estimated_value_numeric); return result; }, [teams, filters]); return { filteredTeams, isLoading, error, filters, hasActiveFilters, setSearch, setMinValue, setMinTitles, resetFilters, retry: loadTeams }; }
Один хук -- всё: загрузка (useCallback + useEffect), фильтрация (useMemo), сортировка, retry. Никакого отдельного useFilters. Паттерн "fat hook" -- удобен для одного экрана.

API: Параллельная загрузка 26 страниц

259 команд на 26 страницах. Стратегия: загрузить первую → узнать total_pages → параллельно загрузить остальные.

Performance
// src/api/teams.ts const BASE_URL = 'https://jsonmock.hackerrank.com/api/football_teams'; export async function fetchAllTeams(): Promise<FootballTeam[]> { const firstResponse = await fetch(BASE_URL); if (!firstResponse.ok) throw new Error(`API error: ${firstResponse.status}`); const firstPage: TeamsApiResponse = await firstResponse.json(); const allTeams: FootballTeam[] = [...firstPage.data]; if (firstPage.total_pages > 1) { const pageNumbers = Array.from( { length: firstPage.total_pages - 1 }, (_, i) => i + 2 ); const responses = await Promise.all( pageNumbers.map(async (page) => { const response = await fetch(`${BASE_URL}?page=${page}`); if (!response.ok) throw new Error(`API error on page ${page}`); const data: TeamsApiResponse = await response.json(); return data.data; }) ); responses.forEach((teams) => allTeams.push(...teams)); } return allTeams; }
Promise.all -- аналог Swift TaskGroup. Все 25 оставшихся страниц загружаются одновременно. Сортировка перенесена в useMemo хука useTeams (не в API layer).

Каждая карточка -- отдельный компонент, обёрнутый в React.memo(). Props -- как @Binding в SwiftUI.

// src/components/TeamCard.tsx interface TeamCardProps { team: FootballTeam; userLocation: Location | null; rank: number; } function TeamCardComponent({ team, userLocation, rank }: TeamCardProps) { return ( <View style={styles.card} accessibilityLabel={`${team.name}, valued at ${team.estimated_value}, ${team.number_of_league_titles_won} league titles`} accessibilityRole="summary"> <View style={styles.header}> <View style={styles.rankBadge}> <Text style={styles.rankText}>{rank}</Text> </View> <View style={styles.headerText}> <Text style={styles.name}>{team.name}</Text> <Text style={styles.league}>{team.league}</Text> </View> <Text style={styles.value}>{formatValue(team.estimated_value_numeric)}</Text> </View> <View style={styles.statsRow}> {/* Titles, UCL, Trophies, Capacity */} </View> <DistanceButton stadiumName={team.stadium_name} userLocation={userLocation} /> </View> ); } export const TeamCard = memo(TeamCardComponent);
memo() = shouldComponentUpdate. Без memo каждый TeamCard ре-рендерится при любом изменении родителя (фильтры, поиск). С memo -- только когда team, userLocation или rank реально изменились. При 259 карточках это критично.

useDebounce: Оптимизация поиска

Отдельный хук для debounce -- generic, переиспользуемый. В index.tsx поиск debounce-ится через этот хук перед передачей в useTeams.

Hooks Performance
// src/hooks/useDebounce.ts export function useDebounce<T>(value: T, delayMs: number = 300): T { const [debouncedValue, setDebouncedValue] = useState<T>(value); useEffect(() => { const timer = setTimeout(() => { setDebouncedValue(value); }, delayMs); return () => clearTimeout(timer); // cleanup при каждом изменении value }, [value, delayMs]); return debouncedValue; } // Использование в app/index.tsx: const [searchInput, setSearchInput] = useState(''); const debouncedSearch = useDebounce(searchInput, 300); React.useEffect(() => { setSearch(debouncedSearch); // обновляет фильтр в useTeams }, [debouncedSearch, setSearch]);
Двойной state паттерн: searchInput обновляет TextInput мгновенно (responsive UI), а debouncedSearch обновляет filteredTeams через 300ms (без лишних пересчётов). В iOS аналог: Combine .debounce(for: 0.3).

// src/components/FilterControls.tsx const VALUE_PRESETS = [ { label: 'Any', value: null }, { label: '$100M+', value: 100_000_000 }, { label: '$500M+', value: 500_000_000 }, { label: '$1B+', value: 1_000_000_000 }, { label: '$2B+', value: 2_000_000_000 }, ]; export function FilterControls({ minValue, onMinValueChange, ... }: Props) { const [isExpanded, setIsExpanded] = useState(false); return ( <View> <TouchableOpacity onPress={() => setIsExpanded(!isExpanded)}> <Text>Filters {hasActiveFilters ? Расстояние: 3 хука в цепочке

useLocation → useStadiumDistance → DistanceButton. По кнопке: GPS → геокодировать стадион → haversine расстояние.

Hooks Architecture
// src/hooks/useLocation.ts -- запрашивает GPS при mount export function useLocation(): UseLocationReturn { useEffect(() => { let cancelled = false; async function requestLocation() { const { status } = await ExpoLocation.requestForegroundPermissionsAsync(); if (status !== 'granted') { /* error */ return; } const position = await ExpoLocation.getCurrentPositionAsync({ accuracy: ExpoLocation.Accuracy.Balanced, }); if (!cancelled) setLocation({ latitude: ..., longitude: ... }); } requestLocation(); return () => { cancelled = true; }; // cleanup! }, []); } // src/hooks/useStadiumDistance.ts -- geocode + haversine export function useStadiumDistance(stadiumName, userLocation) { const mountedRef = useRef(true); useEffect(() => () => { mountedRef.current = false; }, []); const calculate = useCallback(async () => { const coords = await geocodeStadium(stadiumName); // Map cache! if (!mountedRef.current) return; const km = haversineDistance(userLocation.latitude, ...coords.lat, ...); setDistance(formatDistance(km)); }, [stadiumName, userLocation]); }
Три уровня защиты от race conditions: cancelled flag в useLocation, mountedRef в useStadiumDistance, geocodeCache (Map) в geocoding.ts. В iOS: Task cancellation + weak self + NSCache.

Trade-offs (что спросят на review)

Будь готов объяснить архитектурные решения:

"Почему fetch all upfront, а не pagination?"
259 команд = ~100KB. API не поддерживает серверную фильтрацию. При клиентской фильтрации нужны все данные. При 10K+ элементов -- да, нужен серверный фильтр.
"Почему chips, а не slider для value?"
Диапазон $3M-$2.5B нелинейный. На линейном слайдере 90% значений заняли бы 2% полоски. Chips интуитивнее и accessible (VoiceOver friendly).
"Почему нет Redux/Zustand?"
Один экран, 3 фильтра, один массив. useState + useMemo достаточно. Показывает, что ты знаешь, когда НЕ использовать инструмент.
"Почему distance on-demand, а не при загрузке?"
Geocoding 259 стадионов при mount: ~4 минуты при 1 req/sec (Nominatim policy). Плохой UX. Per-button + cache -- правильный баланс.
"Почему useTeams объединяет fetch + filter?"
Один экран, одно состояние. Разделение на useTeams + useFilters добавит prop drilling и усложнит retry. При масштабировании -- разделять.

Debug: Duplicate Keys

В проекте есть баг. FlatList показывает warning в console: "Encountered two children with the same key". Найди и исправь.

Debugging
BUG (app/index.tsx:57-59)
const keyExtractor = useCallback( (item: FootballTeam) => item.name, [] ); // item.name НЕ уникален! // API возвращает 259 команд с 26 страниц. // Одно и то же имя может встречаться // в разных лигах/контекстах. // // React использует key для reconciliation. // Дубликаты ломают обновление списка.
FIX
const keyExtractor = useCallback( (item: FootballTeam, index: number) => `${item.name}-${item.league}-${index}`, [] ); // Композитный ключ: name + league + index // - name + league: семантически уникально // - index: гарантирует уникальность // // Альтернативы: // - Только index (ломает анимации) // - name + league (если правда уникально) // - UUID с бэкенда (нет в этом API)
Почему это важно: Дубликаты ключей приводят к: (1) некорректным обновлениям при фильтрации, (2) потере состояния DistanceButton, (3) "прыжкам" при скролле. В iOS аналог: одинаковые reuseIdentifier для разных типов ячеек.
Как найти: Console warning в Metro bundler → посмотреть keyExtractor → проверить уникальность item.name в API response (открыть https://jsonmock.hackerrank.com/api/football_teams и сравнить страницы).
