🌐 2. API Layer — Загрузка данных

src/api/teams.ts

Загрузка ВСЕХ команд с пагинированного REST API. Ключевой паттерн: параллельные запросы.

// Шаг 1: Загрузить первую страницу → узнать total_pages export async function fetchAllTeams(): Promise<FootballTeam[]> { const firstResponse = await fetch(BASE_URL); const firstPage: TeamsApiResponse = await firstResponse.json(); const allTeams: FootballTeam[] = [...firstPage.data]; // Шаг 2: Создать массив номеров оставшихся страниц [2, 3, 4...] if (firstPage.total_pages > 1) { const pageNumbers = Array.from( { length: firstPage.total_pages - 1 }, (_, i) => i + 2 ); // Шаг 3: Promise.all = ВСЕ запросы ОДНОВРЕМЕННО const responses = await Promise.all( pageNumbers.map(async (page) => { const response = await fetch(`${BASE_URL}?page=${page}`); const data: TeamsApiResponse = await response.json(); return data.data; }) ); // Шаг 4: Собрать все команды в один массив responses.forEach((teams) => allTeams.push(...teams)); } return allTeams; }
Promise.all = Swift withThrowingTaskGroup. Запускает ВСЕ запросы параллельно, ждёт ВСЕ. Если бы последовательно — 4 запроса × 200ms = 800ms. Параллельно — 200ms total.
«Я загружаю первую страницу для получения total_pages, затем остальные параллельно через Promise.all — это как TaskGroup в Swift.»

🗺️ 3. Geocoding + Кеширование

src/api/geocoding.ts

Преобразование названия стадиона в GPS-координаты через Nominatim (OpenStreetMap).

// Кеш на уровне МОДУЛЯ — живёт между ре-рендерами const geocodeCache = new Map<string, { lat: number; lon: number } | null>(); export async function geocodeStadium( stadiumName: string ): Promise<{ lat: number; lon: number } | null> { // 1. Проверить кеш (O(1) lookup) if (geocodeCache.has(stadiumName)) { return geocodeCache.get(stadiumName) ?? null; } // 2. Запрос к Nominatim API const query = encodeURIComponent(`${stadiumName} stadium`); const response = await fetch( `${NOMINATIM_URL}?q=${query}&format=json&limit=1`, { headers: { 'User-Agent': 'OnwardClubsList/1.0' } } // ^^^ Nominatim ТРЕБУЕТ User-Agent, иначе 403 ); const results: NominatimResult[] = await response.json(); if (results.length === 0) { geocodeCache.set(stadiumName, null); // кешируем и null! return null; } const coords = { lat: parseFloat(results[0].lat), // API возвращает строки! lon: parseFloat(results[0].lon), }; geocodeCache.set(stadiumName, coords); return coords; }
Map на уровне модуля = как static var cache: [String: CLLocationCoordinate2D?] — живёт вне React, не вызывает ре-рендер при обновлении.
Кешируем и null результаты — иначе повторные запросы на несуществующие стадионы будут бить по API каждый раз.

🧮 4. Utils — чистые функции

src/utils/distance.ts + format.ts

Никакого React — просто математика и форматирование.

distance.ts — Haversine Formula
// Расстояние между двумя GPS-точками на сфере export function haversineDistance( lat1: number, lon1: number, // пользователь lat2: number, lon2: number // стадион ): number { const R = 6371; // радиус Земли в км const dLat = toRadians(lat2 - lat1); const dLon = toRadians(lon2 - lon1); const a = Math.sin(dLat / 2) ** 2 + Math.cos(toRadians(lat1)) * Math.cos(toRadians(lat2)) * Math.sin(dLon / 2) ** 2; const c = 2 * Math.atan2( Math.sqrt(a), Math.sqrt(1 - a) ); return R * c; // результат в км } // Форматирование для UI export function formatDistance(km: number): string { if (km < 1) return `${Math.round(km * 1000)} m`; if (km < 10) return `${km.toFixed(1)} km`; return `${Math.round(km).toLocaleString()} km`; }
format.ts — Currency
// 2000000000 → "$2.0B" // 200000000 → "$200M" export function formatValue( valueNumeric: number ): string { if (valueNumeric >= 1_000_000_000) { return `$${( valueNumeric / 1_000_000_000 ).toFixed(1)}B`; } if (valueNumeric >= 1_000_000) { return `$${Math.round( valueNumeric / 1_000_000 )}M`; } if (valueNumeric >= 1_000) { return `$${Math.round( valueNumeric / 1_000 )}K`; } return `$${valueNumeric}`; } // 1_000_000 = numeric separator // то же самое что в Swift!
В Swift расстояние считает CLLocation.distance(from:). Мы реализовали Haversine вручную — показывает глубину понимания на интервью.

⏱️ 5. useDebounce — Custom Hook

src/hooks/useDebounce.ts

Задерживает обновление значения. Типичный паттерн для поиска.

export function useDebounce<T>(value: T, delayMs: number = 300): T { // Внутреннее состояние — "отложенное" значение const [debouncedValue, setDebouncedValue] = useState<T>(value); useEffect(() => { // Запустить таймер: обновить через 300ms const timer = setTimeout(() => { setDebouncedValue(value); }, delayMs); // CLEANUP: если value изменился раньше — отменить старый таймер return () => clearTimeout(timer); }, [value, delayMs]); // ^^^^^^^^^^^^^^^ dependency array — перезапускать при изменении return debouncedValue; }

Пошагово:

User types "R" → value="R" → timer starts (300ms) User types "e" → value="Re" → cleanup cancels old timer → new timer (300ms) User types "a" → value="Rea"→ cleanup cancels old timer → new timer (300ms) User stops → .......... → 300ms passes → setDebouncedValue("Rea") → component re-renders → search executes
Swift эквивалент: Combine.debounce(for: .milliseconds(300), scheduler: RunLoop.main). Та же идея — задержать, пока пользователь не перестанет печатать.

🏗️ 6a. useTeams — State + Loading

src/hooks/useTeams.ts

Это ViewModel приложения. Управляет данными, фильтрацией, сортировкой.

export function useTeams(): UseTeamsReturn { // 4 куска состояния (как 4 @Published свойства во ViewModel) const [teams, setTeams] = useState<FootballTeam[]>([]); const [isLoading, setIsLoading] = useState(true); const [error, setError] = useState<string | null>(null); const [filters, setFilters] = useState<Filters>(INITIAL_FILTERS); // Загрузка данных — обёрнута в useCallback чтобы не пересоздавать const loadTeams = useCallback(async () => { setIsLoading(true); setError(null); try { const allTeams = await fetchAllTeams(); // API вызов setTeams(allTeams); } catch (err) { const message = err instanceof Error ? err.message : 'Failed to load teams'; // instanceof = Swift: if let error = err as? Error setError(message); } finally { setIsLoading(false); // как defer { isLoading = false } } }, []); // ^^ пустой массив = функция создаётся ОДИН раз // Запуск при монтировании (= .onAppear) useEffect(() => { loadTeams(); }, [loadTeams]);
useEffect запускается ПОСЛЕ первого рендера (как viewDidAppear), не до. Поэтому isLoading начинается с true — чтобы показать спиннер сразу.

🔍 6b. useTeams — Фильтрация + Сортировка

src/hooks/useTeams.ts — продолжение

Самая важная часть для интервью — useMemo с фильтрами.

// useMemo = computed property С КЕШИРОВАНИЕМ // Пересчитывает ТОЛЬКО когда teams или filters изменились const filteredTeams = useMemo(() => { let result = [...teams]; // КОПИЯ массива! Никогда не мутируй state // Фильтр по поиску if (filters.search.trim()) { const searchLower = filters.search.toLowerCase().trim(); result = result.filter((team) => team.name.toLowerCase().includes(searchLower) ); } // Фильтр по минимальной стоимости if (filters.minValue !== null) { result = result.filter( (team) => team.estimated_value_numeric >= (filters.minValue ?? 0) ); } // Фильтр по минимуму титулов if (filters.minTitles !== null) { result = result.filter( (team) => team.number_of_league_titles_won >= (filters.minTitles ?? 0) ); } // Сортировка: самые дорогие сверху result.sort( (a, b) => b.estimated_value_numeric - a.estimated_value_numeric ); // Отрицательное = a первый, положительное = b первый return result; }, [teams, filters]); // ^^^^^^^^^^^^^^^^ зависимости — при изменении пересчитать
Без useMemo — фильтрация пересчитывалась бы при КАЖДОМ ре-рендере (клик, скролл, что угодно). С useMemo — только когда реально изменились данные или фильтры.
«useMemo здесь как кешированное computed property. Фильтрация и сортировка — O(n log n) операция, и я хочу запускать её только когда входные данные реально изменились.»

🎛️ 6c. useTeams — Сеттеры и Return

src/hooks/useTeams.ts — завершение

Стабильные ссылки на функции через useCallback.

// Каждый сеттер обёрнут в useCallback const setSearch = useCallback((search: string) => { setFilters((prev) => ({ ...prev, search })); // ^^^^^^ functional update: получи предыдущий state → верни новый // ^^^^^^^^^^^ spread: скопируй все поля, замени search }, []); const setMinValue = useCallback((value: number | null) => { setFilters((prev) => ({ ...prev, minValue: value })); }, []); const setMinTitles = useCallback((titles: number | null) => { setFilters((prev) => ({ ...prev, minTitles: titles })); }, []); const resetFilters = useCallback(() => { setFilters(INITIAL_FILTERS); }, []); // Вычисляемое свойство — дешёвое, не нужен useMemo const hasActiveFilters = filters.search !== '' || filters.minValue !== null || filters.minTitles !== null; // Публичный интерфейс хука (как protocol ViewModel) return { filteredTeams, isLoading, error, filters, hasActiveFilters, setSearch, setMinValue, setMinTitles, resetFilters, retry: loadTeams, // переименовали для потребителя };
Зачем useCallback на сеттерах? Если не обернуть — при каждом ре-рендере создаётся НОВАЯ функция → дочерние компоненты думают что props изменились → лишний ре-рендер. useCallback(fn, []) = одна и та же ссылка навсегда.

📍 7. useLocation — GPS

src/hooks/useLocation.ts

Запрос разрешения + получение координат через expo-location.

export function useLocation(): UseLocationReturn { const [location, setLocation] = useState<Location | null>(null); const [error, setError] = useState<string | null>(null); const [isLoading, setIsLoading] = useState(false); useEffect(() => { let cancelled = false; // флаг для cleanup async function requestLocation() { setIsLoading(true); // Запрос разрешения (показывает нативный диалог) const { status } = await ExpoLocation .requestForegroundPermissionsAsync(); // = CLLocationManager().requestWhenInUseAuthorization() if (status !== 'granted') { if (!cancelled) { setError('Location permission denied'); setIsLoading(false); } return; } // Получить текущую позицию const position = await ExpoLocation .getCurrentPositionAsync({ accuracy: ExpoLocation.Accuracy.Balanced, // GPS + Wi-Fi + cell towers // Хороший баланс точности и батареи }); if (!cancelled) { setLocation({ latitude: position.coords.latitude, longitude: position.coords.longitude, }); } } requestLocation(); // Cleanup — если компонент размонтируется во время запроса return () => { cancelled = true; }; // = task.cancel() в .onDisappear }, []); // пустые deps = один раз при монтировании return { location, error, isLoading }; }
Паттерн let cancelled = false — стандарт React для async операций которые нельзя abort'нуть. Без него — React warning: «Can't update state on unmounted component».

📐 8. useStadiumDistance — On-Demand

src/hooks/useStadiumDistance.ts

Считает расстояние по нажатию кнопки, не автоматически. Экономит батарею и трафик.

export function useStadiumDistance( stadiumName: string, userLocation: Location | null ): UseStadiumDistanceReturn { const [distance, setDistance] = useState<string | null>(null); const [isLoading, setIsLoading] = useState(false); const [error, setError] = useState<string | null>(null); // useRef = хранит значение БЕЗ перерисовки (как stored property) const mountedRef = useRef(true); useEffect(() => { return () => { mountedRef.current = false; }; }, []); // calculate — вызывается при тапе на кнопку const calculate = useCallback(async () => { if (!userLocation) { setError('Location not available'); return; } setIsLoading(true); setError(null); const coords = await geocodeStadium(stadiumName); // может вернуть из кеша (мгновенно) или с сети if (!mountedRef.current) return; // unmounted? if (!coords) { setError('Stadium not found'); return; } const km = haversineDistance( userLocation.latitude, userLocation.longitude, coords.lat, coords.lon ); setDistance(formatDistance(km)); // "1,234 km" }, [stadiumName, userLocation]); return { distance, isLoading, error, calculate }; }
«Расстояние считается lazy — только по нажатию. 24 команды × геокодинг = 24 запроса на загрузке, плюс Nominatim имеет rate limit. Lazy подход экономит батарею и трафик.»

Без memo
User types in search → filteredTeams changes → FlatList re-renders → ALL 10 visible cards re-render → even if their team data is identical → WASTED renders ❌
С memo
User types in search → filteredTeams changes → FlatList re-renders → memo() compares prev props vs new props → only cards with DIFFERENT props re-render → EFFICIENT renders ✅
Swift эквивалент: Equatable conformance на SwiftUI View. memo() делает shallow comparison всех props — если ссылки/значения те же, пропускает ре-рендер.

<FlatList data={filteredTeams} // массив данных renderItem={renderItem} // как cellForRowAt keyExtractor={keyExtractor} // уникальный ID {/* Поиск + фильтры ВЫШЕ списка, скроллятся вместе */} ListHeaderComponent={ <> {/* Fragment — группа без View */} <SearchBar value={searchInput} onChangeText={setSearchInput} /> <FilterControls minValue={filters.minValue} minTitles={filters.minTitles} onMinValueChange={setMinValue} onMinTitlesChange={setMinTitles} onReset={handleReset} hasActiveFilters={hasActiveFilters} /> </> } {/* Пустой список → спиннер или EmptyState */} ListEmptyComponent={ isLoading ? <ActivityIndicator /> : <EmptyState hasFilters={hasActiveFilters} onReset={handleReset} /> } {/* PERFORMANCE TUNING */} initialNumToRender={10} // первый render: 10 items maxToRenderPerBatch={10} // макс 10 items за кадр скролла windowSize={5} // держать 5 "экранов" в памяти removeClippedSubviews={true}// убирать из нативного дерева />
UITableView с estimatedRowHeight + prefetchDataSource. FlatList рендерит только видимые + буфер. Без этих настроек — все 24 item'а рендерятся сразу (ок для 24, плохо для 10,000).
«FlatList — это виртуализированный список. initialNumToRender контролирует первый paint, windowSize — сколько экранов буферизовать, removeClippedSubviews отсоединяет невидимые view от нативного дерева.»
